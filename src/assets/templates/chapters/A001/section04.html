Dentro de los objetivos de las pruebas tenemos:

  1. Disminución del Preproceso de corrección de Defectos
  2. Apropiación de la cultura de calidad por parte del equipo humano
  3. Medición y control del producto durante la etapa de desarrollo de Software
  4. Analisis de causas de defectos para apoyar la mejora de procesos de Software

Como principales razónes de por que son necesarias la pruebas de Software tenemos las siguientes:

  1. Mejoran la Calidad: Las pruebas como fase de un proceso de desarrollo de Software, mejoran la calidad del producto final. Los defectos existen en el software por varios motivos: El software está escrito por humanos, los cuales cometen errores, presuponen cosas, pueden interpretar situaciones incorrectamente… Tiempo para el desarrollo muy reducido. Condiciones ambientales; un producto puede funcionar muy bien en las condiciones donde se esté desarrollando, pero cuando se migra a un escenario más parecido al de producción pueden salir fallos originados únicamente por el entorno de ejecución. Y tales defectos existirán siempre por mucho que se pruebe y depure el producto tal y como dijo Dijktra en 1970.
  2. Ahorrar Costes: Una de las razones por la que es necesario probar es que eliminar un defecto de software cuesta mucho más cuanto más tarde se detecte. Si no se prueba durante el proceso de desarrollo de software, encontrar un error en producción puede hacer que el producto no sea rentable. Así de sencillo.
